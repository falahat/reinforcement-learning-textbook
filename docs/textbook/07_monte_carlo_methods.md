## 5.1 Why MC after DP

Chapter 6's DP assumed:

1. Known $P$ and $R$.
2. Enumerable state space.

Monte Carlo methods relax assumption 1: **estimate value functions from
sampled trajectories**. They keep assumption 2 (still tabular). Removing
assumption 2 needs function approximation (Chapter 10).

The relaxation is huge: the agent now learns from *interaction*, not from
a model. This is the fundamental shift to "real" reinforcement learning.

### When MC is right

- You can sample trajectories (real or simulated rollouts).
- Episodes terminate.
- You want unbiased value estimates.
- Variance is acceptable.

### When MC is wrong

- Continuing tasks with no episode boundary (use TD).
- High variance is unacceptable (use TD or actor-critic).
- You need updates more often than once per episode (use TD).

## 5.2 First-visit and every-visit MC for prediction

**Problem:** estimate $V^\pi(s)$ from trajectories generated by following
$\pi$.

**Idea:** $V^\pi(s) = \mathbb{E}[G_t \mid s_t = s]$. If we sample many
trajectories starting from $s$, the average of observed returns
$\hat{V}(s) \approx V^\pi(s)$.

### First-visit MC

For each episode, record the return $G$ obtained from each state's
*first* occurrence. Average across episodes.

```python
def first_visit_mc(pi, num_episodes, gamma):
    returns = defaultdict(list)
    V = defaultdict(float)
    for ep in range(num_episodes):
        trajectory = generate_episode(pi)  # [(s_0, a_0, r_0), ...]
        G = 0
        visited = set()
        # Iterate trajectory backwards
        for t in range(len(trajectory) - 1, -1, -1):
            s, a, r = trajectory[t]
            G = r + gamma * G
            if s not in visited:
                visited.add(s)
                returns[s].append(G)
                V[s] = np.mean(returns[s])
    return V
```

### Every-visit MC

Same, but record returns from *every* visit to a state in an episode, not
just the first.

```python
def every_visit_mc(pi, num_episodes, gamma):
    returns = defaultdict(list)
    V = defaultdict(float)
    for ep in range(num_episodes):
        trajectory = generate_episode(pi)
        G = 0
        for t in range(len(trajectory) - 1, -1, -1):
            s, a, r = trajectory[t]
            G = r + gamma * G
            returns[s].append(G)
            V[s] = np.mean(returns[s])
    return V
```

### Bias-variance tradeoff

| | First-visit MC | Every-visit MC |
|---|---|---|
| Bias | Unbiased | Slightly biased (correlated samples from same episode) |
| Variance | Higher | Lower |
| Convergence | $\to V^\pi$ as $n \to \infty$ | $\to V^\pi$ as $n \to \infty$ |
| Theory | Cleaner | Slightly more involved |

Both converge to $V^\pi$. In practice they're almost indistinguishable
for typical problems. First-visit is the cleaner choice for theoretical
work; every-visit is slightly easier to implement (no `visited` set).

### Incremental updates

Maintaining $\hat{V}(s)$ as a running average lets you avoid storing all
returns:

$$
\hat{V}_{n+1}(s) = \hat{V}_n(s) + \frac{1}{n+1}\big(G - \hat{V}_n(s)\big)
$$

For non-stationary settings, use a fixed step size:

$$
\hat{V}_{n+1}(s) = \hat{V}_n(s) + \alpha \big(G - \hat{V}_n(s)\big)
$$

The constant $\alpha$ version tracks moving averages — useful when $\pi$
changes (as in control), where the "true" $V^\pi$ is shifting.

## 5.3 MC for control

Estimating $V^\pi$ is **policy evaluation**. The harder problem is
**policy improvement** without a model.

### Why $Q^\pi$, not $V^\pi$?

Recall from Chapter 5: with $V^\pi$, picking the greedy action requires
$\arg\max_a \mathbb{E}[r + \gamma V(s')]$, which needs $P$. With $Q^\pi$,
just $\arg\max_a Q(s, a)$ — no model needed.

**MC control estimates $Q^\pi$ instead of $V^\pi$.** Then greedification
is model-free.

### MC with exploring starts

The simplest MC control algorithm:

```
1. Initialize Q(s, a) arbitrarily, π greedy w.r.t. Q.
2. Loop:
   a. Pick (s_0, a_0) uniformly at random (exploring start).
   b. Generate episode following π from (s_0, a_0).
   c. For each (s, a) in episode (first-visit), update Q(s, a) toward G.
   d. π ← greedy(Q).
```

**Exploring starts** guarantees every $(s, a)$ pair is visited
infinitely often, which is needed for $Q$ to converge for all pairs.

This works in simulators where you can choose starting states. **Doesn't
work in the real world or in the Simulator**, where you can't ask the
environment to start in an arbitrary state.

### $\epsilon$-soft policies

Alternative: use an $\epsilon$-soft policy — one that picks every action
with probability at least $\epsilon / |\mathcal{A}|$.

**$\epsilon$-greedy** is a simple form: with probability $1 - \epsilon$, pick
$\arg\max_a Q(s, a)$; with probability $\epsilon$, pick uniformly.

```python
def epsilon_greedy(Q, s, epsilon, num_actions):
    if np.random.random() < epsilon:
        return np.random.randint(num_actions)
    return np.argmax(Q[s])
```

Under $\epsilon$-soft policies, every state-action pair gets visited
infinitely often (as long as every state is reachable), so $Q$ converges
for *the $\epsilon$-greedy policy*. The optimal $\epsilon$-greedy policy is
not the same as the unconstrained optimal policy — but for small $\epsilon$,
it's close.

### GLIE: Greedy in the Limit with Infinite Exploration

To converge to the truly optimal policy, decay $\epsilon$ over time:
$\epsilon_n \to 0$ slowly enough that $\sum \epsilon_n = \infty$. This is
the **GLIE** condition (Greedy in the Limit with Infinite Exploration).
Typical choice: $\epsilon_n = 1/n$ or $\epsilon_n = 1/\sqrt{n}$.

Under GLIE + Robbins-Monro on the step size, MC control converges to
$\pi^{\star}$.

## 5.4 Off-policy MC: importance sampling

### Why this section is the technical heart of off-policy RL

The conceptual problem this section solves is the most important
practical question in model-free RL: **can I use data collected by
one policy to learn about a different policy?** If yes, you can
explore with $\epsilon$-greedy and learn the greedy policy; you can
reuse historical experience after a policy update; you can train
from human demonstrations. If no, every policy change throws away
all previous data.

Importance sampling (IS) is the *unbiased* answer: yes you can,
provided you reweight by $\pi/\mu$. But "yes" is misleading because
the reweighting *increases variance multiplicatively per step* —
and trajectories have many steps. The variance can be so bad that
"unbiased" estimates with 200 trajectories give worse predictions
than a biased on-policy estimator with 20.

The structural awareness of "unbiased but huge variance" is what
drives the whole off-policy story across the book: weighted IS
(§5.4 below), Q-learning's max-trick (Ch8 §6.4 — implicit IS that
sidesteps the trajectory product), V-trace and Retrace (Ch11 §9.6,
Ch13), and ultimately the deadly triad (Ch17 — what happens when
off-policy + bootstrap + FA collide). Read this section as the
*source* of the practical compromises later chapters make.

### Setting

Sometimes you want to learn about one policy (the **target policy**
$\pi$) while following another (the **behavior policy** $\mu$). For
example:

- Learn about the greedy policy while exploring with
  $\epsilon$-greedy.
- Reuse data collected under an old policy after the policy has
  been updated.
- Learn from demonstrations (behavior policy = human or expert).

This is **off-policy learning**.

### Deriving the importance sampling ratio

If we want $\mathbb{E}_\pi[X]$ but our samples come from $\mu$, we
can reweight. The derivation has three steps, each cleanly justified:

$$
\mathbb{E} _\pi[X] \stackrel{(1)}{=} \sum_x x \cdot P _\pi(x) \stackrel{(2)}{=} \sum_x x \cdot \frac{P _\pi(x)}{P _\mu(x)} \cdot P _\mu(x) \stackrel{(3)}{=} \mathbb{E} _\mu\left[\frac{P _\pi(X)}{P _\mu(X)} \cdot X\right].
$$

Step (1) is the definition of expectation under $\pi$. Step (2)
multiplies and divides by $P_\mu$ — a legal manoeuvre because both
$P_\pi$ and $P_\mu$ are positive distributions; this is also where
the requirement "$\mu(x) > 0$ wherever $\pi(x) > 0$" comes from
(otherwise the ratio is undefined). Step (3) re-recognises the sum
as an expectation under $\mu$.

The ratio $\rho = P_\pi / P_\mu$ is the **importance sampling
ratio**. It is the "exchange rate" between samples from $\mu$ and
samples from $\pi$.

**Why this is conceptually huge.** We've just shown that *any
expectation* under $\pi$ can be computed from samples drawn under
$\mu$, as long as we know both densities. The agent doesn't have to
*follow* $\pi$ to *learn about* $\pi$. That's the whole bedrock of
off-policy learning.

For a trajectory $\tau = (s_0, a_0, s_1, a_1, \ldots)$:

$$
\rho(\tau) = \prod_t \frac{\pi(a_t \mid s_t)}{\mu(a_t \mid s_t)}
$$

(Transition probabilities $P(s_{t+1} \mid s_t, a_t)$ cancel since they're the same
under both policies.)

### Ordinary IS for MC

To estimate $V^\pi(s)$ from $\mu$-sampled trajectories:

$$
\hat{V}^\pi(s) = \frac{1}{N(s)} \sum_{i} \rho_i \cdot G_i
$$

where the sum is over trajectories that visit $s$ and $G_i, \rho_i$ are
the return and IS ratio from $s$ in trajectory $i$.

This is **unbiased** but **variance can be huge** — $\rho$ is a product
of many ratios, and can blow up.

Length $L$ multiplies the effect — each additional step is one more factor

### Weighted IS

A biased but lower-variance alternative:

$$
\hat{V}^\pi(s) = \frac{\sum_i \rho_i G_i}{\sum_i \rho_i}
$$

The normalization in the denominator keeps things bounded. Asymptotically
unbiased, with bounded variance.

**Modern practice:** almost always use weighted IS. Ordinary IS is
theoretically pretty but practically catastrophic.

### What IS doesn't say

- **It doesn't fix the variance problem, only the bias problem.**
  The reweighting makes the estimator unbiased; it doesn't make it
  low-variance. With long trajectories, $\rho$ is a product of many
  ratios, and a single rare event makes one ratio huge.
- **It requires absolute continuity: $\mu(x) > 0$ wherever $\pi(x) > 0$.**
  If the behaviour policy can't possibly produce a trajectory the
  target policy would have produced, the reweighting fails. This
  excludes learning a deterministic target $\pi$ from a deterministic
  behaviour $\mu \neq \pi$ — at any state where they pick differently,
  $\rho = \infty$. The practical fix is $\epsilon$-greedy behaviour
  policies.
- **It scales exponentially with trajectory length.** The variance
  of $\rho(\tau) = \prod_t \pi/\mu$ grows multiplicatively with $T$.
  After 100 steps with typical $\pi/\mu$ ratios near 2-3, $\rho$ can
  be $10^{30}$ or $10^{-30}$. Per-decision IS (below) trims this.

### Per-decision IS [Precup, Sutton & Singh 2000]

The trajectory's full $\rho$ is a product of many terms — variance scales
exponentially in trajectory length. **Per-decision IS** truncates the
product, applying $\rho$ only to ratios that matter for each reward.
Substantially lower variance for long trajectories.

[S&B 2018, Sec. 5.9] has the full derivation; [Precup, Sutton & Singh 2000]
is the foundational paper. We won't dwell on this — it matters more for
TD methods (Chapter 8's off-policy story).

## 5.5 MC vs DP

| | DP | Monte Carlo |
|---|---|---|
| Requires model | Yes | No |
| Requires episodes | No | Yes |
| Bootstraps? | Yes | No |
| Bias | None | None (first-visit) |
| Variance | None | High |
| Per-step cost | $O(\|\mathcal{A}\|\|\mathcal{S}\|)$ | $O(1)$ |
| Total samples needed | 0 (compute directly) | Many trajectories |

The DP-MC contrast frames the rest of RL: **TD methods (Chapter 8)
combine the best of both** — they don't need a model (like MC) but they
bootstrap (like DP), giving lower variance and online updates.

## 5.6 Variance: why MC is hard in practice

Suppose returns $G$ have variance $\sigma^2$. After $n$ episodes, the
variance of the sample mean is $\sigma^2 / n$. To estimate $V$ to within
$\epsilon$ with confidence $1-\delta$, you need

$$
n \geq \frac{\sigma^2 \log(2/\delta)}{2 \epsilon^2}
$$

(by a Bernstein bound or Hoeffding if returns are bounded). For typical
RL problems with large rewards and discounting:

- $\sigma^2$ can easily be 1-1000.
- $\epsilon = 0.01$ requires $\sim 10^4 / \epsilon^2 = 10^8$ episodes.

This is why pure MC is rarely used in modern RL. Almost everything is
TD-based — same data, much lower variance.

## 5.7 The Simulator basically doesn't use MC

The Simulator's TD-based learner (Chapter 8) is the rule for one
reason: **episodes don't really exist**. The agent's "life" continues
indefinitely (until death, but death is rare and most learning happens
mid-life). There's no natural episode boundary at which to compute $G$.

This is why MC is a *teaching* algorithm here — useful for understanding
the foundations, but TD is what gets used.

### Where MC-like thinking does appear

- **Calibration runs** like `td_error_noise_floor_deep_calibration` in
  [`chaos_and_sabotage.rs`](https://github.com/falahat/simulator/blob/main/crates/sim/app/tests/pathologies/chaos_and_sabotage.rs)
  average TD error over a trailing window — this is *batch* averaging, a
  cousin of MC.
- **Cross-seed assertions** in tests like [`hungry_consume.rs`](https://github.com/falahat/simulator/blob/main/crates/sim/app/tests/tasks/hungry_consume.rs)
  effectively use multiple "samples" (different seeds) to estimate
  metrics. Same statistical idea.
- **Return-decomposition methods like RUDDER** ([Chapter 19](19_long_horizon_credit.md))
  do something MC-like: predict full episode returns and decompose
  them. But they require episodic structure that the Simulator lacks.

## 5.8 Worked example: blackjack

Reproduced from [S&B 2018, Sec. 5.1]. State = (player sum, dealer's
upcard, usable ace). Actions = {hit, stick}. Reward = win/lose/draw at
end of episode. Episode = one hand.

First-visit MC with random initial policy "hit if sum < 20" converges to
$V$ for that policy in ~10000 episodes. MC control with exploring starts
converges to the optimal hit/stick policy.

The takeaway: **MC is conceptually clean, works fine on small toy
problems, but scales poorly to anything with high variance or no episode
structure**.

## 5.9 Project tie-in

### Where MC is not used (and why)

The Simulator's [`learning_rate.rs`](https://github.com/falahat/simulator/blob/main/crates/engine/q_learning/src/learning_rate.rs)
implements TD(0), not MC. The reasons:

1. **No episode boundaries** — the agent's life doesn't have natural
   resets.
2. **Online updates needed** — the agent has to act every cognition
   tick; can't wait for an episode end.
3. **Lower variance** — TD's bootstrapping reduces variance by orders of
   magnitude (Chapter 8).

### Where MC concepts do appear

- **TD-error band measurement** — see
  [`crates/sim/sim_test_harness/src/calibration.rs`](https://github.com/falahat/simulator/blob/main/crates/sim/sim_test_harness/src/calibration.rs)'s
  `trailing_window_stats`. This averages TD errors over a window — a
  small-scale MC-style estimate.
- **The `RewardAttributionLedger`** in `learning_homeostatic.rs` tracks
  total credit attributed per action over the run — similar in spirit to
  MC's "total return from each first-visit."

### What would change if you used MC

Imagine you wanted to use MC instead of TD in the Simulator. You'd need:

1. **Define episodes.** Maybe "from one hunger crisis to the next?"
   "From birth to death?" Each choice changes what's learned.
2. **Store the full trajectory.** Memory: 251-dim observation × thousands
   of ticks × hundreds of episodes = a lot.
3. **Update only at episode end.** Cognition becomes batch — agent
   doesn't update during the trajectory.

This would change the Simulator's character substantially. Not impossible
— some research projects use MC for grounded agents — but a different
beast.
